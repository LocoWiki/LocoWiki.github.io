Workflow

本地维护建议

如果你不是一次性下载，而是准备持续跟仓库同步，建议按这个顺序处理。

先克隆仓库

把完整目录结构拉到本地，后续更新时只需要增量拉取。

再处理同步脚本

需要同步 Wiki 或批量抓取资料时，再进入脚本和自动化说明。

最后回到文档阅读

文档页负责检索和目录跳转，仓库目录负责原始文件管理，两者不要混用。

Policy

大文件与批量抓取

下载策略要区分“仓库原始文件”“公开可抓取内容”和“超大文件管理”三种情况。

大文件说明

单文件超过约 100MB 时建议使用 Git LFS（或替换为合规外链）。若你发现下载速度慢，优先使用 git clone 或镜像。

镜像与公开来源

论文和外部资源优先保留索引与合法来源；真正需要批量抓取时，再在本地执行脚本。

论文 PDF 批量抓取（可公开获取）

如果你在本地维护仓库，推荐用脚本批量下载（优先 arXiv / OpenReview 等开源来源）：

python3 scripts/fetch_open_access_pdfs.py --download
