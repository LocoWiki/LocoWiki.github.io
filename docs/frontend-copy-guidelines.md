# 前端文案规范

## 1. 命名统一

- 前台统一使用“飞书知识库”，不再使用历史命名 `Readmydock`
- 技术名词优先中文表达，必要时在括号中补充英文

## 2. 文案分层

- 用户层：说明“是什么、怎么用、遇到问题怎么办”
- 开发层：部署、脚本、事件链路、调试说明放到 `docs/`

## 3. 提示语风格

- 面向用户：简洁、可执行、少术语
- 面向开发：明确路径、命令、依赖关系

## 4. 状态面板文案

- 显示“最近同步时间”与简要说明
- 避免展示测试事件、调试消息、内部动作名

## 5. 变更自检清单

- 全仓检索是否仍出现旧命名（`Readmydock`）
- 检查前台页面是否出现维护者专属说明
- 检查 `README.md` 与 `docs/` 链接是否有效
